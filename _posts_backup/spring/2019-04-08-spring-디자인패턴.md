#### Proxy 패턴
* Spring AOP
* 실제 작업하는 객체를 감싸서 실제 객체를 요청하기 전이나 후에 인가 처리, 생성 자원이 많이 드는 작업에 대해 백그라운드 처리, 원격 메소드를 호출하기 위한 작업 등을 하는데 사용한다.
* VPN
* 흐름 제어만 할 뿐, 결과값을 조작하면 안된다.

#### Singleton 패턴
* Spring Bean
* 최초 한번만 메모리를 할당하고 그 메모리에 인스턴스를 만들어 사용한다.
* 여러번 호출 되더라도 새로 생성하는 것이 아니고 객체를 반환한다.
* 메모리 낭비 방지
* 전역 인스턴스이기 때문에 데이터 공유가 쉽다.
* 멀티 쓰레드 환경에서 동기화 처리를 해줘야 한다.

#### Template Method 패턴
* Spring RestTemplate
* 전체적인 레이아웃을 통일시키지만 상속받은 클래스로 하여금 유연함을 주도록 하는 패턴

#### MVC 패턴
* Spring MVC

#### Circuit breaker 패턴
* Spring Hystrix
* 서비스 A와 서비스 B 중간에 서킷 브레이커 설치
* 서비스 B로의 모든 호출은 서킷 브레이커를 통하게 되고, 정상적인 상황에서는 문제없이 통과한다.
* 서비스 B가 문제가 생겼을 때에는 강제로 연결을 끊어 서비스 A에서 스레드들이 더 이상 요청을 기다리지 않도록 해서 장애가 전파하는 것을 막는다.

#### Fall-back 패턴
* Spring Hystrix
* 장애가 발생했을 때 대신 실행할 로직을 미리 구현해 놓고 장애 발생시 해당 로직을 실행한다.
